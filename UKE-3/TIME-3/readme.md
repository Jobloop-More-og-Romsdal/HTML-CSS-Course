# W3C HTML Validator:
[W3C HTML VALIDATOR](https://validator.w3.org/)
- Link: https://validator.w3.org/
   
**Du kan validere ved å:**
- Lime inn koden din
- Laste opp en fil
- Eller gi den en URL

Den er litt streng, men på en "jeg vil bare at du skal gjøre det riktig"-måte.

# Wave – WebAIM
[WebAIM](https://wave.webaim.org/)
- Link: https://wave.webaim.org/

er et verktøy som også analyserer tilgjengelighet, ikke bare syntaks.

Det viser kontrastfeil, manglende alt-tekster, feil heading-struktur og alt annet som kan påvirke tilgjengeligheten på siden din.

Den gir også visuell markering på selve nettsiden, noe som gjør det superlett å forstå hva som må fikses.

# Lighthouse (Google)
[Lighthouse](https://chromewebstore.google.com/detail/lighthouse/blipmdconlkpinefehnmjammfjpmpbjk?pli=1)
- Link: https://chromewebstore.google.com/detail/lighthouse/blipmdconlkpinefehnmjammfjpmpbjk?pli=1

Links to an external site.går bredere til verks og analyserer:
- Ytelse
- Tilgjengelighet
- SEO (Search engine optimization, eller søkemotoroptimalisering)
- Best practices

Lighthouse kan legges til som extension i de fleste nettlesere, og du kjører den via tilleggsmenyen i nettleseren din..

# Universell utforming og Uutilsynet  
**Hvorfor dette er viktig**
I Norge er universell utforming ikke bare «nice to have», det er faktisk et krav for offentlige virksomheter og mange private.

[Uutilsynet](https://www.uutilsynet.no/)
(Digitaliseringsdirektoratet)
- Link: https://www.uutilsynet.no/

fører tilsyn med UU-kravene og gjennomfører kontroller av   tilgjengeligheten på nettsteder.

Dårlig tilgjengelighet kan føre til både:
- Manglende brukervennlighet
- Lovbrudd
- Og i verste fall pålegg eller gebyr

Verktøy som W3C-validator, Wave og Lighthouse er derfor ikke bare for perfeksjonister, men for å sikre at alle faktisk kan bruke det du lager – uansett forutsetninger
